## Context

- 使用者想把 origin/main 的新 commits 同步到本機
- 本機可能有未 commit 的修改，不能被覆蓋
- 策略：stash 本地修改 → fetch + rebase → stash pop

## Your Task

安全同步 origin/main 到本機，保留本地所有修改。

### Steps

1. **確認目前 git 狀態**
   ```bash
   git status --short
   git log --oneline -5
   ```
   顯示目前 branch、有哪些修改、目前在哪個 commit。

2. **檢查 origin/main 有多少新 commits**
   ```bash
   git fetch origin
   git log --oneline HEAD..origin/main
   ```
   如果沒有新 commits，告知使用者「已是最新」，結束。

3. **如果有未 commit 的修改，自動 stash**
   ```bash
   git stash push --include-untracked -m "safe-pull: auto stash before rebase $(date +%Y%m%d-%H%M%S)"
   ```
   記錄是否有 stash（影響後續步驟）。

4. **Rebase onto origin/main**
   ```bash
   git rebase origin/main
   ```
   如果 rebase 發生衝突：
   - 顯示哪些檔案衝突
   - 告知使用者手動解決後執行 `git rebase --continue`
   - 不要自動 abort，等待使用者決定

5. **如果之前有 stash，還原回來**
   ```bash
   git stash pop
   ```
   如果 stash pop 有衝突，顯示衝突檔案並告知使用者。

6. **最終報告**
   顯示：
   - 同步了幾個新 commits
   - 本地修改是否成功還原
   - 目前 git status

### Guardrails

- 如果目前不在 git repo 中，立即告知並停止
- 如果目前 branch 是 main/master，提醒使用者是否確定要直接在主幹 rebase
- 不要強制 push，不要修改任何遠端狀態
- stash 使用有意義的 message 以便追蹤
